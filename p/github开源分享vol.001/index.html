开源模型汇总

  • 背景趋势：国外模型在闭源（“庇员”），而国内加速开源（“拼命开源”），近期因世界人工智能大会集中爆发。
  • 阿里模型系列：
    • 7月21日：开源千万325B A22（QianWan3家族基础模型），刷新多项记录。
    • 7月22日：开源千万3Coder 480B，配合千万Coder工具（命令行AI开发助手，对标CoderCoder），建议通过千万Coder体验效果。
    • 7月25日：开源千万325B A222Sinking（最新思考模型）。
    • 7月28日：开源1.2.2（业界首个Moe架构视频声称基础模型），包括文上视频T2V、涂声视频i2V和统一视频声称i2T2V；业界广泛基于此微调。
    • 7月30-31日：开源千万330B A3B和sinking版本，提供小规模本地部署选项。
  • 其他机构模型：
    • 7月26日：上海人工智能实验室开源Intern SE（235B QianWan3 + 6B视觉编码器），强化科学领域训练，支持分子式、地震信号等数据理解。
    • 7月27日：同学会员开源三地世界模型（业界首个可成绩式蔓游可交互世界生成模型），用于游戏开发和数据内容创作。
    • 7月28日：制补开源GRM4.5系列（GRM4.5335B、A22B、i2等），社区反响好，冲到Hugging Face榜首。
    • 7月30日：困难万为开源Skywalking Unipake 1.5B（多么太统一模型），整合图像理解、生成和编辑能力。
    • 8月1日：监约星辰发布Snap3（新一代主力机座模型），宣称推里效率最高。

Coze的开源版本

  • 背景：Code开源智能体版本，缩小与Dify和N8n的差距
  • 部署要求：至少两核4G内存，建议更高配置；组件设计清晰，服务组件化明确。
  • 功能对比：
    • 优势：架构简洁，易于持续发展。
    • 不足：插件、工作流和用户管理相对欠缺，可能是为企业版预留空间或暂未发布。
  • 建议：根据需求选择Code、Dify或n8n，三者差异不大。

开源HR管理系统

  • 项目：Fripper HRMS（源自ERPNX HR模块，V14起独立运营）。
  • 功能模块：包含13个模块，覆盖员工管理、入职、修架、爆销、技效、新资、税务等；需配置和初始化。
  • 部署与使用：
    • 支持Docker部署，上手方便。
    • 提供功能完整的移动应用程序。
  • 注意事项：税务模块需验证是否适配国内税制（本地特色强）。
  • 适用场景：小公司可试用，强调现代潜医使用。

多功能文件分享工具

  • 项目：Copy Party（多功能文件服务器）。
  • 功能：
    • 支持浏览器、WaveDive或FTP访问。
    • 操作友好：拖转文件上传、批量重命名、文件搜索、上传回撤、时间自动消回、缩略图、音频编码。
    • 文档全面，功能丰富。
  • 部署：
    • Windows：直接执行ESC文件。
    • 其他平台：运行Python命令，默认端口80/443。
  • 使用场景：分享大文件（如10GB视频），避免网盘限速，确保私有化本地保存。

开源简历优化工具

  • 项目：Resume Master（ATS通过神器），通过内相工程定制简历。
  • 功能：改造简历以通过ATS筛选，非创建简历。
  • 部署与运行：
    • 环境：Python 3.8+、Node.js。
    • 步骤：执行npm run build，使用本地olama大模型（数据本地运行）。
  • 使用流程：上传简历和JD（职位描述），系统提供定制建议。
  • 问题：测试中因Pantetic数据验证失败（JD不符要求），可能需优化适配。

最后分享资料

  • 云计算蓝皮书2025：新统院出品，覆盖20年云计算历史，强调AI时代云计算价值。
  • 北京大学内部资料：DeepSeek研讨资料（非最新但全面），适合直接用于讲课PVT。

总结

本期聚焦GitHub热点，突出国内开源大模型爆发（如阿里QianWan系列、Intern SE等），覆盖AI开发工具（Coze开源版）、企业系统（Fripper HRMS）、实用工具（Copy Party文件分享、Resume Master简历优化）及行业资料（云计算蓝皮书、北大DeepSeek研讨）。核心趋势是国内开源加速，工具强调本地部署与私有化，建议开发者根据需求试用模型和小型工具，注意税务适配及部署配置。
