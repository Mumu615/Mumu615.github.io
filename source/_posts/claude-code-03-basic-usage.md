安装配置完成之后，最重要的不是继续研究理论，而是立刻让 Claude Code 做一个小任务。

哪怕只是一个番茄钟、待办清单、Markdown 整理脚本，都可以。你需要先建立体感：它不是聊天机器人，而是会读文件、改文件、跑命令的本地 Agent。

## 第一个任务：让它做一个番茄钟

在项目目录打开终端，输入：

```bash
claude
```

进入 Claude Code 之后，可以先给它一个非常简单的任务：

```text
帮我做一个简单的番茄钟应用。
```

Claude Code 很可能不会立刻动手，而是先问你：

- 想用什么技术栈
- 是否需要网页界面
- 是否需要声音提醒
- 是否要保存设置

这很正常。

Claude Code 比较主动。当它认为上下文不够清楚时，会先问问题，或者进入计划模式。

## 理解 Plan Mode：计划模式

Claude Code 有一个很重要的模式：**Plan Mode**。

在计划模式下，它不会直接修改文件，而是先给你一份计划。

比如它会写：

- 准备创建哪些文件
- 使用什么框架
- 页面结构怎么设计
- 状态如何管理
- 最后如何运行

你确认后，它才会开始执行。

这对小白非常重要。

因为你不一定看得懂所有代码，但你至少可以先看懂“它打算做什么”。

## 三种常见权限模式

Claude Code 里可以通过 `Shift + Tab` 切换权限模式。

常见体验大概分三类。

## 默认模式

默认模式下，Claude Code 会自己判断哪些事情可以做，哪些事情需要问你。

适合新手。

因为它比较稳，不会一上来就疯狂改文件或跑命令。

## 自动编辑模式

这个模式下，它可以更主动地编辑文件。

但涉及运行终端命令，比如安装依赖、删除文件、启动服务时，通常还是会征求你的同意。

适合你已经信任它的当前计划，但仍希望控制命令执行。

## 危险权限模式

有些用户会用更激进的启动参数，让 Claude Code 一路绿灯自动执行。

这个模式效率很高，但风险也更高。

新手不建议一开始就用。

等你熟悉 Git、回滚、权限和项目结构之后，再考虑是否使用。

## 为什么它还会问能不能运行命令？

很多新手会遇到这种情况：你已经同意它修改文件了，但它安装依赖时又来问你。

原因是：**编辑文件和执行终端命令是两类权限。**

比如它执行：

```bash
npm install
```

这不是编辑文件，而是在你的电脑上运行命令、下载依赖。

所以它会再次确认。

通常你会看到几个选项：

- 只同意这一次
- 同意，并以后在当前项目里不再询问类似命令
- 不同意，继续讨论

新手建议选择“只同意这一次”，慢慢熟悉它在做什么。

## 怎么运行 Claude Code 给你的命令？

当 Claude Code 做完项目后，可能会告诉你运行：

```bash
npm run dev
```

你有两种方式执行。

## 新开一个终端运行

这是最推荐的方式。

在 IDE 里新开一个终端，执行它给你的命令。

这样 Claude Code 的对话终端不会被占住。

## 在 Claude Code 中用感叹号运行

Claude Code 支持一种 Bash Mode。

你可以在命令前加 `!`：

```bash
!npm run dev
```

这表示你不是在和 Claude Code 对话，而是在让它执行一条终端命令。

但如果这个命令是长期运行的，比如启动开发服务器，就可能阻塞对话。

这时更建议新开终端。

## 基础交互方式一：直接打字

最简单：

```text
帮我把这个页面改得更适合移动端。
```

适合简单任务。

但如果任务复杂，只打一句话通常不够。Claude Code 会花更多 token 去探索，也更容易跑偏。

## 基础交互方式二：@ 文件

当你明确知道要改哪个文件时，可以直接 @ 它。

比如：

```text
@src/App.tsx 请解释这个文件的代码逻辑，并用中文补充必要注释。
```

这样做的好处是更精准，也更省 token。

Claude Code 不会每次都把整个项目塞进上下文。它通常会通过搜索、读取、分析来逐步理解项目。

如果你已经知道关键文件，直接 @ 文件能减少它乱翻项目的成本。

## 基础交互方式三：用文档写长提示词

终端里编辑长提示词很痛苦。

所以你可以新建一个 Markdown 文件，比如：

```text
requirements.md
```

把详细需求写进去，然后告诉 Claude Code：

```text
请阅读 @requirements.md，并严格按照里面的要求实现。
```

这是非常推荐的工作方式。

提示词越具体，Claude Code 执行越靠谱，也越省上下文。

## 基础交互方式四：发图片

Claude Code 也支持图片输入。

你可以把参考图拖进对话，或者粘贴截图，让它根据图片做：

- UI 还原
- 配色参考
- 截图问题分析
- 报错界面识别
- 设计稿转代码

一图胜千言。尤其做前端时，图片比你用文字描述“高级感”“科技感”“不要太土”有效得多。

## 基础使用的核心原则

刚开始不要追求复杂玩法。

先把这个闭环跑通：

1. 给任务
2. 看计划
3. 同意执行
4. 运行项目
5. 看效果
6. 让它修改
7. 再验证

只要这个闭环顺了，你就已经真正入门了。
