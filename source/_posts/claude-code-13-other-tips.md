## 新手最容易踩的坑

## 不存档就乱改

这是最大的问题。

用 Claude Code 做项目，一定要学会 Git 存档。

否则 AI 改坏了，你会不知道怎么回去。

Claude Code 自带 `/rewind`，但它不是万能的。

真正可靠的后悔药是 Git。

## 一个会话聊到底

长时间不 `/compact`，上下文会越来越乱。

每完成一个阶段，就压缩一次。

不要等模型变慢、变笨、答非所问时才想起来管理上下文。

## 需求太短

“帮我做个高级感网页”这种提示词太虚。

你越不说清楚，它越要自己探索，反而更费 token，也更容易跑偏。

更好的方式是写一个需求文档，然后 @ 给它。

## 乱装 MCP 和插件

新手最容易看到什么都想装。

但工具越多，维护成本越高，也可能让上下文变复杂。

先围绕真实需求装。

## 把 API Key 暴露给别人

API Key 是你的调用凭证。

不要发截图，不要提交到 Git，不要粘贴到公开网页。

需要求助时，把 Key 打码。

## 推荐的新手学习路线

如果你是第一次学 Claude Code，建议按这个顺序练：

1. 安装 Claude Code
2. 配置模型或 API
3. 在空文件夹里做一个简单项目，比如番茄钟
4. 学会 `Shift + Tab` 切换模式
5. 学会 @ 文件和 @ 需求文档
6. 学会新开终端运行项目
7. 学会 Git 存档
8. 学会 `/compact` 管理上下文
9. 学会创建项目级 `CLAUDE.md`
10. 再学习 Skill、MCP、CLI、Sub Agent、Hook

不要一开始就追求高级玩法。

先把基础工作流跑顺，你就已经超过大多数只收藏不动手的人了。

## 一套实用的 Claude Code 工作流

日常使用时，可以按这个节奏来。

## 开始任务前

先写清楚目标：

```text
我要做什么？
最终结果是什么？
有哪些限制？
怎么验证完成？
```

如果任务复杂，就写进 `requirements.md`。

## 执行任务时

让 Claude Code 先计划：

```text
请先阅读需求，给出实现计划。暂时不要修改文件。
```

确认计划后再执行。

## 完成阶段后

让它验证：

```text
请运行必要的检查，确认当前版本可以正常运行。
```

然后提交 Git：

```text
当前版本我满意，请帮我提交一个 Git 存档。
```

## 长任务中

阶段性压缩上下文：

```text
/compact
```

## 项目稳定后

更新 `CLAUDE.md`：

```text
请根据这次开发中踩过的坑，更新项目 CLAUDE.md，让以后不要再犯同样的问题。
```

这套流程比“想到什么就问什么”可靠得多。

## 你真正要学的不是某个按钮

Claude Code 的重点不是多会一个命令。

真正重要的是，你开始从“一问一答地使用 AI”，转向“构建系统，让 AI 自己推进任务”。

这意味着你要学会：

- 给清晰目标
- 提供准确上下文
- 让它先计划
- 用 Git 管住风险
- 用文档沉淀规则
- 用 Skill 和工具扩展能力
- 用子 Agent 并行处理复杂任务

当你习惯这种协作方式后，未来再出现新的 Agent、新的 CLI、新的模型，你都会上手得更快。

## 最后总结

这个系列从 0 到 1 梳理了 Claude Code 的完整使用路径。

你现在应该知道：

- Claude Code 是运行在本地终端里的 AI Agent
- 它能读写文件、运行命令、制定计划并执行任务
- 它之所以好用，不只是因为模型强，也因为 Harness 做得好
- 新手可以搭配 Cursor、Trae 或 VS Code 使用
- 模型可以通过官方订阅，也可以通过 API 和 CC Switch 配置
- `Shift + Tab` 可以切换权限模式
- @ 文件、图片、需求文档，都是给上下文的好方法
- `/compact`、`/clear`、`/context`、`/rewind` 是常用命令
- Claude Code 自带回滚不够，真正可靠的是 Git
- `CLAUDE.md` 和 Auto Memory 可以让它更懂你
- Skill、MCP、CLI、Sub Agent、Hook、插件，是它变强的关键扩展能力

如果你只记住一句话，那就是：

**不要把 Claude Code 当成一个聊天框，要把它当成一个可以被训练、被约束、被扩展、被纳入你工作流的本地 Agent。**

从今天开始，你不需要一次掌握所有高级功能。

先安装，先跑起来，先让它帮你完成一个真实的小任务。

真正的门槛不是技术，而是开始动手。
